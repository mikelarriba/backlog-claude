## Feature Title
Tracetool 2.0 — Controlled, Self-Sufficient Analysis & Report Export

## Context
Tracetool is MIDAS's integrated analysis capability that generates and exports reports from test data. In its current form, the feature relies heavily on external dependencies outside the team's control — creating fragility and unpredictable failure modes whenever those dependencies change or become unavailable. Additionally, Tracetool configuration requires direct database intervention, which bypasses governance, introduces human error risk, and creates a bottleneck that only developers can resolve. As MIDAS matures toward V2 and the team assumes greater ownership of its platform, this dependency exposure and operational gap must be addressed to ensure Tracetool is stable, maintainable, and self-sufficient.

## Objective
By the end of this Feature's Epics, Tracetool's external dependency surface is reduced and explicitly managed, all Tracetool configuration is manageable through a product interface without direct database access, and the report export pipeline meets a defined reliability SLA — eliminating unplanned downtime caused by external failures or manual DB intervention.

## Value
- **For Test Engineers:** Reliable report exports with fewer unexplained failures and no need to raise support tickets for configuration changes.
- **For Data Engineers:** Full visibility and control over Tracetool configuration through a governed interface, removing the need for ad-hoc DB queries and reducing the risk of misconfiguration.
- **Business impact:** Reduces operational risk and developer toil, accelerates incident resolution, and positions Tracetool as a stable, owned capability within the MIDAS platform rather than a fragile external dependency chain.

## Execution
Planned Epics that will deliver this Feature (each will be linked via "Is Contained" in JIRA):

1. **Epic:** Tracetool Dependency Audit & Decoupling — Map all external dependencies, assess risk, and introduce abstraction or fallback layers to give the team control over failure modes.
2. **Epic:** Tracetool Configuration Management — Replace direct database configuration with a governed admin interface (UI + API) for managing Tracetool settings without developer intervention.
3. **Epic:** Report Export Pipeline Stabilisation — Harden the export flow end-to-end, define a reliability baseline, and instrument monitoring and alerting for proactive failure detection.

## Out of Scope
- Infrastructure provisioning or environment changes (flag as dependency to the infra team)
- Adding new report types or extending Tracetool's analytical capabilities — this Feature focuses on stability and control, not feature expansion
- Full V2 async rewrite of Tracetool — V2 migration is a future consideration once the current pipeline is stabilised
- Access control or permission changes to who can use Tracetool
